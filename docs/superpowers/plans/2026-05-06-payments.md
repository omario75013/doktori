### Task 3 тАФ `lib/bank-transfer.ts` + tests

**Files**:
- Create: `apps/web/lib/bank-transfer.ts`
- Create: `apps/web/lib/bank-transfer.test.ts`

API:
```ts
import "server-only";

export interface BankTransferConfig {
  iban: string;
  bic?: string;
  bankName: string;
  accountHolder: string;
}

export async function createBankTransferIntent(args: {
  appointmentId: string;
  patientId: string;
  doctorId: string;
  amount: number;
  expiryDays: number;  // from platformSettings.payment.bank_transfer.expiry_days
}): Promise<{
  intentId: string;
  reference: string;
  amount: number;
  bankConfig: BankTransferConfig;
  expiresAt: Date;
}>;

export async function confirmBankTransfer(args: {
  intentId: string;
  adminId: string;
}): Promise<void>;

export async function rejectBankTransfer(args: {
  intentId: string;
  adminId: string;
  reason: string;
}): Promise<void>;
```

`createBankTransferIntent` reads doctor's `doctor_payment_methods` row for `bank_transfer` method, fetches IBAN/BIC/etc from `config` JSON. Generates a 12-char reference (alphanumeric, easy to read), inserts row with status=pending.

`confirmBankTransfer` runs in transaction:
- Update intent status=confirmed, confirmed_by, confirmed_at
- Update appointment payment_status=paid, payment_method=bank_transfer
- Insert wallet_transactions row (credit doctor)

Tests with vitest + mock DB / a real test DB (uses CI Postgres):
- createBankTransferIntent generates unique ref + correct expiry
- createBankTransferIntent fails if doctor has no bank_transfer method enabled
- confirmBankTransfer is idempotent (calling twice doesn't double-credit)
- rejectBankTransfer doesn't credit wallet

Commit: `feat(bank-transfer): manual verification flow lib + tests (Phase 2 #2)`

### Task 6 тАФ Stripe webhook handler

**File**:
- Create: `apps/web/app/api/payments/stripe/webhook/route.ts`

POST handler:
1. Read raw body (NOT JSON.parse тАФ Stripe needs raw)
2. Get `Stripe-Signature` header
3. `verifyWebhookSignature(body, sig)` тЖТ throws if invalid тЖТ return 400
4. Insert into `stripe_events_log` with ON CONFLICT DO NOTHING (idempotency)
5. Switch on `event.type`:
   - `checkout.session.completed` тЖТ mark appointment paid
   - `payment_intent.succeeded` тЖТ idempotent same-as-above
   - `charge.refunded` тЖТ mark refunded
   - others тЖТ ignore
6. Return 200

Important: this route MUST NOT be cached; `export const dynamic = 'force-dynamic'`.

Commit: `feat(api/payments): Stripe webhook handler with replay protection`
